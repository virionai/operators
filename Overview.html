Capsules.run Operators

An offline local-first investigative workspace for Gemma4Good/Kaggle, with optional local Gemma reasoning through Ollama and portable Capsule handoff.

Run

./run-local.sh from the operators folder. The app opens at http://127.0.0.1:5174 and expects Node >=20.

Local Runtime

Live Gemma answers use Ollama at http://127.0.0.1:11434/api/chat with gemma4:latest. If Ollama is disabled or unreachable, deterministic offline analysis keeps the app self-contained with no cloud dependency.

Primary Surfaces

  • Top HUD for runtime identity, inference target, session state, and operator provenance.
  • Blank spatial workspace for local evidence, generated surfaces, chronology, topology, and document inspection.
  • Gemma resident assistant for contextual Q&A, tasks, and communication plans.
  • Tiny bottom analysis queue for active local model work.
  • Ledger Events surface for append-only context continuity and selected evidence capture.

Interactive Features

  • Workspace tabs, workflow modes, canvas modules, document search, local uploads, and attachment selection.
  • Highlight text anywhere in the workspace and add it to Context Continuity.
  • Ask Gemma questions, cancel inference, toggle the optional local runtime, and export the local Capsule handoff.

Status

The documentation set reflects the current runnable local app, including one-line startup, Ollama/Gemma expectations, offline constraints, event ledger naming, and known local limits.

Known Limits

The export is a hashed local JSON artifact with capsule_version: "0.6-local-ui", not byte-compatible Capsule v0.6. Signing, evidence parsing, and chain-of-custody behavior remain integration gates.
